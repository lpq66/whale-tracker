### Key Pattern Identified
Position size relative to wallet matters A LOT:
- WIFCAT: 195 SOL wallet bought 21.72 SOL (11%) → WIN
- Losers: 1055 SOL wallets buying 3 SOL (0.3%) → DUST, filtered out

### Filters Implemented (config.yaml)
1. **Position size ≥ 1% of wallet** — kills dust buys
2. **MC range $50K - $500K** — skip dead or too-big tokens
3. **Min liquidity ratio ≥ 5%** — ensure exit liquidity
4. **Token dedup DISABLED** — multiple whales buying same token is a signal, not noise
5. **All alerts logged to `all_alerts` table** — for pattern analysis even if filtered out
